# 文件系统

## 1. 三层模型

1. **文件系统核心层（独立 crate）**
   - 只处理磁盘格式与算法（如 superblock/BPB、目录项、分配表、簇链）。
   - 不依赖内核对象（`DEntry/Inode/Process`）。

2. **内核适配层（kernel/fs/<name>）**
   - 把核心层能力映射到 VFS 抽象（`DirInodeBackend/BytesInodeBackend`）。
   - 负责权限语义、inode 元数据同步、页读写对接等内核语义。

3. **VFS 组装层（kernel/fs/mod.rs）**
   - 统一管理挂载、路径解析、文件系统类型分发。
   - 不关心某个 FS 的磁盘细节。

文件系统不应认识具体设备驱动，只依赖最小设备接口（如 `BlockDevice` trait）。

这样依赖方向固定为，`kernel` -> 具体 fs 实现

## 2. 可扩展性

如果边界清晰，新增文件系统（如 ext2）只需：
- 新增一个核心 crate
- 新增一个 kernel 适配层
- 在 VFS 注册类型与构造入口

现有 FAT32 的拆分就是这一路线的具体实践。
